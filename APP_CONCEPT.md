# Budget Buddy — App Concept Reference

> مرجع داخلي لفكرة التطبيق يُرجع إليه قبل بناء أي feature جديدة.

---

## Core Idea

تطبيق لمتابعة المصاريف الشهرية بشكل منظم.
المستخدم يعرف فلوسه راحت فين، من غير ما تظهر له أي قيمة بالسالب، وكل حاجة مبنية على الواقع.

**المبدأ الأساسي:** لا سالب أبداً — لو الرصيد خلص، التطبيق يعرض بديلاً، مش يسمح بتجاوز.

---

## User Journey

### 1. الإعداد الأولي (Onboarding)
- المستخدم يدخل **العملة** اللي بيشتغل بيها
- يدخل **المرتب الشهري** (مثلاً 1000 دولار)

### 2. توزيع الميزانية (Budget Allocation)
- يروح على صفحة فيها **الفئات الرئيسية** (Categories)
- يوزع المرتب على الفئات

**قواعد التوزيع:**

| الحالة | ما يحدث |
|--------|---------|
| حاول يوزع أكتر من المرتب | Alert Dialog: "مش معاك كفاية" → يعرض عليه يوزع الباقي أو يعدل الأرقام |
| وزع أقل من المرتب (مثلاً 950 من 1000) | رسالة: "عندك 50 متبقية — تضيفهم للـ Saving ولا توزعهم؟" |
| وزع المرتب كله بالظبط | تمام، يكمل |

### 3. تسجيل المصاريف (Expense Logging)
- المستخدم يسجل مصروف ويربطه بفئة
- المبلغ يتخصم من رصيد الفئة المختارة

**قواعد الخصم:**

| الحالة | ما يحدث |
|--------|---------|
| المصروف ≤ رصيد الفئة | يتخصم عادي |
| المصروف > رصيد الفئة | يتخصم اللي موجود بس → Alert: "مش معاك كفاية في الفئة دي" → يعرض عليه ياخد الفرق من فئة تانية |
| قبل ياخد الفرق من فئة تانية | يعرض عليه الفئات اللي فيها المبلغ المطلوب أو أكتر فقط → يختار → يتخصم الفرق منها |

## Categories System

### الفئات الافتراضية
- Housing (إيجار / سكن)
- Food & Drinks (أكل وشرب)
- Transportation (مواصلات)
- Healthcare (صحة)
- Entertainment (ترفيه)
- Other (متنوع)
- **Saving (ادخار)** ← فئة أساسية دايماً موجودة، دايماً آخر واحد في الترتيب

### فئات مخصصة
المستخدم يقدر يضيف فئات إضافية حسب احتياجه.

### Subcategories
كل فئة رئيسية ممكن تتقسم لـ subcategories لتفصيل أكتر (مثلاً "Food" → "مطاعم", "سوبرماركت").

---

## Core Rules (لا تُكسر أبداً)

1. **لا سالب أبداً** — أي فئة رصيدها لا يقل عن صفر
2. **التوزيع لا يتعدى المرتب** — مجموع الـ allocations = المرتب أو أقل
3. **الفرق دايماً يتوجه للـ Saving** — لو في مبلغ مش موزع، يُعرض إضافته للادخار
4. **الخصم من فئة تانية باختيار المستخدم** — مش تلقائي، المستخدم هو اللي يقرر

## Data Model (Overview)

```
UserInfo
  └── currency        العملة المختارة
  └── monthlySalary   المرتب الشهري

Category
  └── name            اسم الفئة
  └── allocatedAmount المبلغ المخصص من المرتب
  └── spentAmount     المنصرف الفعلي
  └── color / icon    مظهر الفئة

Subcategory
  └── parentCategoryId FK → Category
  └── name / color / icon / spentAmount

Transaction
  └── categoryId      FK → Category
  └── amount          المبلغ
  └── date            التاريخ
  └── note            ملاحظة اختيارية
```

**Progress per Category:**
```
progress  = spentAmount / allocatedAmount  (0.0 → 1.0)
remaining = allocatedAmount - spentAmount  (لا تقل عن 0)
```
